## üéØ Descripci√≥n del Proyecto

Este proyecto es la segunda fase del an√°lisis de **TelecomX LATAM**, centrado en el desarrollo de un pipeline de Machine Learning para **predecir la evasi√≥n de clientes (churn)**. ## üìà Principales Hallazgos del Modelo

Tras comparar varios algoritmos, el modelo de **Regresi√≥n Log√≠stica**, entrenado sobre datos balanceados con SMOTE, fue seleccionado como el de mejor rendimiento general.

1. **üèÜ Rendimiento del Modelo:** El modelo de Regresi√≥n Log√≠stica alcanz√≥ un **ROC-AUC de 0.84** y un **Recall del 73%** para la clase "Churn". Esto significa que es altamente efectivo para identificar clientes en riesgo.
2. **üìâ Principal Factor de Retenci√≥n: `tenure` (Antig√ºedad):** Es, con diferencia, el predictor m√°s fuerte. A mayor antig√ºedad del cliente, menor es su probabilidad de evasi√≥n.
3. **üìù Contratos a Largo Plazo:** Tener un contrato de `one_year` o, especialmente, de `two_year`, reduce dr√°sticamente la probabilidad de churn. El contrato `month-to-month` es un fuerte indicador de riesgo.
4. **üåê Internet de Fibra √ìptica:** De manera contraintuitiva, contratar `internetservice_fiber_optic` est√° positivamente correlacionado con el churn, sugiriendo que las expectativas de los clientes podr√≠an no estar siendo satisfechas.
5. **üõ°Ô∏è Servicios de Soporte y Seguridad:** La ausencia de `onlinesecurity` y `techsupport` son factores importantes que aumentan la probabilidad de evasi√≥n.

## üõ†Ô∏è Metodolog√≠a

El proyecto sigue un pipeline de Machine Learning estructurado para garantizar la robustez y reproducibilidad de los resultados.

```mermaid
graph LR
    A[üßπ Preprocesamiento y Encoding] --> B[‚öñÔ∏è Manejo de Desbalance con SMOTE];
    B --> C[‚öôÔ∏è Entrenamiento de Modelos];
    C --> D[üìä Evaluaci√≥n y Selecci√≥n];
    D --> E[üß† Interpretabilidad con SHAP];
    E --> F[üöÄ Recomendaciones Estrat√©gicas];
```

**Preprocesamiento:** Las variables categ√≥ricas se transformaron mediante One-Hot Encoding y las num√©ricas se escalaron con StandardScaler.

**Manejo de Desbalance:** Se aplic√≥ SMOTE al conjunto de entrenamiento para crear un dataset balanceado, asegurando que el modelo no ignore a la clase minoritaria (clientes que cancelan).

**Entrenamiento y Evaluaci√≥n:** Se entrenaron y compararon los modelos LogisticRegression, RandomForestClassifier y HistGradientBoostingClassifier usando m√©tricas como Precisi√≥n, Recall, F1-Score y ROC-AUC.

**Selecci√≥n de Modelo:** Se eligi√≥ el modelo con el mejor balance entre rendimiento t√©cnico (ROC-AUC) y relevancia para el negocio (Recall).

**Interpretabilidad:** Se utilizaron Permutation Importance y los gr√°ficos de SHAP para entender el impacto y la direcci√≥n de cada variable en las predicciones del modelo.
